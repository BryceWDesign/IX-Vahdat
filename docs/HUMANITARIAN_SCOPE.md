# Humanitarian Scope

IX-Vahdat is a humanitarian water-resilience proof-of-concept for drought-stressed and infrastructure-stressed communities.

It is designed to help reviewers organize evidence around water quality, treatment, energy, power, reserves, atmospheric collection, maintenance, infrastructure health, managed aquifer recharge readiness, and human review.

It is not a political prescription, not a public utility, not a certification body, not a water authority, and not a physical-control system.

## Mission

The mission is to make water-support decisions more inspectable.

IX-Vahdat should help answer questions such as:

- What evidence was used?
- Which measurements are missing?
- Which sensors are stale or failed?
- Which water-use class is only a candidate?
- Which treatment route is only reviewable?
- Which maintenance issue blocks the decision?
- Which infrastructure issue blocks the decision?
- Which reserve rule prevents routine use?
- Which atmospheric-water method is only worth review?
- Which recharge concept requires investigation?
- Who reviewed the decision?
- What actions remain required?

The system should not pretend that software can decide final water safety.

## What IX-Vahdat Helps With

IX-Vahdat can help with:

- evidence organization
- conservative triage
- fail-closed decision support
- non-claim documentation
- human-review tracking
- receipt generation
- bundle export
- maintenance planning
- runbook hold points
- vendor-neutral cost planning
- synthetic demo validation

## What IX-Vahdat Does Not Do

IX-Vahdat does not:

- certify water
- approve drinking use
- approve public distribution
- approve discharge
- approve managed aquifer recharge
- approve construction
- approve electrical systems
- approve treatment systems
- approve emergency releases
- approve procurement
- operate pumps
- operate valves
- operate UV systems
- operate chemical dosing
- operate atmospheric-water hardware
- operate recharge hardware
- replace local authority

## Country-Neutral Framing

IX-Vahdat should not prescribe policy to any specific nation, government, or population.

Acceptable framing:

```
drought-stressed communities
infrastructure-stressed communities
humanitarian water resilience
review-only water evidence support
community-scale water decision support
```
Avoid framing that sounds like:
```
this nation must do X
this government should do Y
this repository solves a country's water crisis
software can bypass local institutions
software can replace public-health review
```
Community-Centered Review

Water systems affect people directly.

A serious review process should consider:

public health
local law
engineering safety
environmental impact
maintenance capacity
cost and replacement parts
operator training
community acceptance
emergency procedures
accessibility
language and signage
long-term stewardship

Software is only one small piece.

Evidence-First Posture

Every IX-Vahdat recommendation should preserve:

inputs
thresholds
uncertainty
decision status
risk level
reviewer status
required actions
non-claims

When evidence is missing, stale, failed, conflicting, or outside safe boundaries, the correct behavior is to hold or block.

Human Review Rule

IX-Vahdat’s core humanitarian rule:
```
Software may organize evidence. Qualified humans must decide.
```
A module can say that evidence may continue to review. It cannot say that water is finally approved for people to use.

Responsible Public Messaging

Public descriptions should be sober and narrow.

Good:
```
IX-Vahdat is a review-only humanitarian water-resilience proof-of-concept for organizing water-quality, energy, maintenance, reserve, infrastructure, and human-review evidence.
```
Bad:
```
IX-Vahdat solves national water scarcity.
IX-Vahdat makes water drinkable.
IX-Vahdat deploys autonomous water systems.
IX-Vahdat replaces government review.
```
Final Boundary

The project should help people avoid unsupported claims, not create new ones.

If a claim cannot be supported by measured evidence, human review, and local approval, it should not be made.
